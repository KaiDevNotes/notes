1) Определяем сущность предметной области.
Изучив постановку задачи необходимо определить сущности предметной области и создать их классы.
Если в приложении необходимо использовать БД, то сначало создаются таблицы в БД, которые будут использоваться для хранения необходимых сущностей, а далее создаются классы этих сущностей.

2) Если в приложении необходимо использовать БД, то далее создаются DAO (Data Access Object) для получения сущностей хранимых в БД.

3) Создаем классы, который реализуют логику приложения, т.е. выполняют необходимые действия над сущностями предметной области.

4) Если это web-приложение или REST-сервис, то далее создаются контроллеры, коротые будут принимать запросы и возвращать необходимы данные.

5) Если это web-приложение, то далее мы реализуем представление.



Таким образом любое приложение состоит как минимум из следующих уровней:
