НАТИВНАЯ УПАКОВКА

"Нативная упаковка" (Native Packaging) или "Автономная упаковка приложения" (Self-Containde Application Package) - это пакет, содержащий вместе java-приложение и среду выполнения Java для конкретной платформы. Т.е. в результате мы получим инсталятор, запустив который пользователь получит приложение состоящее из самого приложения и среды (JRE) его исполнения.

После инталяции файлы приложения будут распологаться в директории [disk]:/Users/[yourname]/AppData/Local/[app name]

Это один из способов РАЗВЕРТЫВАНИЯ приложения.
Развертывание - это процесс упаковки и доставки програмного обеспечения к пользователю.

NetBeans версии 8.0

----------------------------------------------------------------------------------------------

1) Включаем в NetBeans возможность упаковывать проект в инсталяционный файл (.exe, .msi и тд в зависимости от операционной системы в которой происходит сборка).

(Right-click) на проекте -> Properties -> Deployment -> устанавливаем checkbox "Enable Native Packaging"

Теперь, когда мы делаем (Right-click) на проекте, мы можем видеть пункт меню "Package as"

----------------------------------------------------------------------------------------------

2) Для того, чтобы NetBeans мог собирать проект в инсталяционный файл необходимо установить на ПК приложение Inno Setup 5. Также в системной переменной PATH необходимо добавить путь к папке, в кот. установлено приложения Inno Setup 5.

Мой компьютер -> (Right Click) Свойства -> Дополнительные параметры системы -> Переменные среды

----------------------------------------------------------------------------------------------

3) Чтобы добавить иконку к инсталяционному exe-файлу и exe-файлу для запуска приложения, необходимо добавить путь к иконке в:

(Right-click) на проекте -> Properties -> Deployment -> Common Deployment Properties -> Edit... 

Иконка должна иметь расширение .ico или .icon

----------------------------------------------------------------------------------------------

4) Выполняем нативную упаковку:

(Right-click) на проекте -> Package as -> EXE Installer

После этого в папке [директория проекта]\dist\bundles получаем exe-файл инсталятора приложения.

После установки приложения по данному инсталятору в директории [disk]:/Users/[yourname]/AppData/Local/[app name] появятся файлы приложения, среди которых будет exe-файл для запуска приложения.




ПРИМЕЧАНИЕ:

Если в проекте используются дополнительные ресурсы, например, картинки, то путь к ним в приложении необходимо формировать как показано в примере.

Допустим нам необходимо установить иконку icon.png для окна JavaFX-приложения. Иконка находится в пакете com.example.resources. В этом случаем в приложении для установки данной иконки необходимо использовать следующий код:

String iconPath = this.getClass().getResource("/com/example/resources/icon.png").toString();       
this.primaryStage.getIcons().add(new Image(iconPath));
